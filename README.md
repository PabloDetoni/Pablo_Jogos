A ideia principal do meu projeto é a de um site de jogos, onde usuários produzem informações diversas, ligadas a um banco de dados, onde um adm pode lidar com elas.
O meu projeto está dividido em 3 principais partes: “backend”, “frontend” e “documentação”; No Backend lido com a parte lógica que está  ligada ao banco de dados, onde a comunicação e funções são usadas para inserir algo no banco de dados. O frontend lida com os jogos presentes no site, onde o usuário pode interagir (jogando, observando dados e alterando eles caso seja ADM). Por fim, a documentação tem informações como a ideia principal do projeto, banco de dados e informações usadas (schema que deve ser inserido no banco) locais onde estão descritos a organização e dados usados para a base do sistema. Com o meu projeto consegui entender de maneira geral como se é a utilização de um banco de dados implementado num site, mesmo sendo apenas uma versão didática, organização e divisão dos dados e funções dos arquivos foi essencial pro meu projeto funcionar, pois ao deixar bem claro o que cada coisa faz e sua função, consigo lidar mais facilmente com problemas e encontrar soluções no meu projeto.
No projeto, está presente 3 principais telas: 
1. Login onde um usuário se cadastrar ou entrar, se já cadastrado.
2. A tela principal onde os jogos estão presentes e as opções de logout, visualização de rank e estatísticas (um com dados locais e o outro vindo do banco de dados, o rank).
3. Painel de administração, exclusivo do ADM, dividido em Dashboard (Para visualização geral de tudo que está presente no projeto), Gestão (Local para gerenciar os dados do projeto, utilizando CRUDs distintos para isso) Visualização (Onde eu vejo as informações presentes e produzidas pelo projeto) e Ferramentas (Onde coisas extras estão no projeto, como backup e reset do sistema).
A criação do projeto começou simples porém com o tempo ele foi adquirindo características um pouco mais complexas, mesmo não sendo exatamente o que eu buscava consegui entregar cerca de 80% do que imaginava nos primeiros meses, coisas como um perfil, foto e um modelo pra amizades e recuperação de senha não foram implementadas, mas eram ideias que eu gostaria de ter colocado no projeto. No caso da experiência com a IA percebi que no meu caso é muito importante comunicação de uma forma extensa e aos poucos, pois assim consigo explicar e obter os resultados que busco, pois estou lidando com poucas alterações feitas da IA e acho mais fácil.
Tenho uma preferência pela parte do backend pois acho mais interessante lidar com essa parte lógica, de comunicação de dados dentro do sistema, onde um usuário não precisa necessariamente mexer.
Sobre a experiência que tive no projeto, achei muito divertido e consegui entender de forma mais conceitual funções relacionadas aos jogos e a lógica por trás delas,  consegui ter uma noção melhor em como posso lidar com variáveis globais e como padronizar funções quando usadas em vários locais.
Então de forma de resumida, com meu projeto aprendi como banco de dados, backend e frontend deve estar dividido e o que ocorre em cada um. Então essa divisão quando de forma completa e bem estruturada faz com que um projeto tenha um desempenho superior a um completamente compactado, que era o caso do meu anterior (server muito extenso causava muitos problemas, mas dividi todo o meu projeto em pequenas partes e lido melhor com elas).
